# Async Test Suite - 异步功能测试套件

## 模块职责

异步功能测试套件专门用于测试 Sparreal OS 的异步任务执行器、调度器和相关功能，验证异步运行时的正确性和性能。

## 入口与启动

### 测试入口
- **`src/main.rs`**: 异步测试主程序
- **配置文件**: 支持多架构的 QEMU 配置

### 配置文件
- `qemu-aarch64.toml`: AArch64 架构的 QEMU 配置
- `qemu-la64.toml`: LoongArch64 架构的 QEMU 配置
- `loongarch64.toml`: LoongArch64 特定配置

## 对外接口

### 依赖的内核接口
- **异步运行时**: `sparreal_rt::os::async::*` - 异步任务执行器
- **定时器**: `sparreal_rt::os::time` - 异步定时器
- **日志**: `log` crate - 测试日志输出

## 关键依赖与配置

### 测试依赖
```toml
[dependencies]
log = {workspace = true}          # 日志接口
sparreal-rt = {workspace = true}  # Sparreal 运行时
```

## 测试内容

### 主要测试场景
1. **基础异步任务**: 简单的异步任务创建和执行
2. **任务调度**: 多个任务的调度和执行顺序
3. **定时器测试**: 异步定时器的正确工作
4. **并发测试**: 多个异步任务并发执行
5. **资源管理**: 异步资源的安全使用和清理

### 性能测试
- **调度延迟**: 测量任务调度的时间延迟
- **内存使用**: 监控异步运行时的内存占用
- **吞吐量**: 测试系统在大量异步任务下的性能

## 常见问题 (FAQ)

### Q: 如何添加新的异步测试？
A: 在 `src/` 中添加新的测试函数，使用 `#[test]` 标记，并集成到主测试流程中。

### Q: 测试失败如何调试？
A: 检查日志输出，确认异步任务的状态和调度顺序。

### Q: 如何测试性能？
A: 使用计时器测量关键操作的时间，并记录性能指标。

### Q: 多架构测试如何配置？
A: 使用相应的 QEMU 配置文件，确保每个架构的正确运行。
