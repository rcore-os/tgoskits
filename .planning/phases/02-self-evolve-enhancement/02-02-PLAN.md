**Subsection A: Cross-Plugin Skill Reference Validation (per D-02, SE-02)**

This check scans ALL agent files (`.claude/agents/*.md`), not just self-evolve. The logic is:

1. Parse `~/.claude/plugins/installed_plugins.json` to enumerate all available `plugin:skill` combinations. Use `os.path.expanduser("~/.claude/plugins/installed_plugins.json")` — do NOT use raw tilde in paths (RESEARCH.md Pitfall 3).

2. Scan all `.claude/agents/*.md` files for `plugin:skill` references. Extract references from both frontmatter `skills:` lists and body text `skill-name` mentions. A regex like `([a-z][\w-]+:[a-z][\w-]+)` captures plugin-prefixed skill names.

3. Cross-reference: for each reference found in agent files, check if it exists in the installed_plugins.json enumeration. Flag missing references as WARN severity.

4. The check uses Python inline or a bash invocation that reads installed_plugins.json. Document the approach following the RESEARCH.md Pattern 2 code example (use `json.load()`, iterate `data["plugins"]` keys, scan `skills/` subdirectories for `SKILL.md` existence).

5. For project-local skills (no `plugin:` prefix), skip this check — they are validated by the existing D4 checklist items against `.claude/skills/` directory.

**Subsection B: Agent-Name Collision Detection (per D-04, SE-06)**

This check scans globally installed plugin agent directories and compares names against the 6 TGOSKits agent names. Follow RESEARCH.md Pattern 3:

1. TGOSKits agent names (hardcoded list): `pr-review`, `test-gen`, `bug-hunt`, `driver-audit`, `impl`, `self-evolve`.

2. Scan all globally installed plugins' `agents/` directories from installed_plugins.json entries. For each `agent_file` found, extract `basename(agent_file).replace(".md", "")` and compare with exact string equality against the TGOSKits list. Use exact basename match — NOT substring/grep (RESEARCH.md Pitfall 5: avoid partial matches like "review" matching both "pr-review" and "code-reviewer").

3. Result: WARNING only (non-blocking per D-04). Surface in the D4 report. If zero collisions found (expected — RESEARCH.md confirmed this), report: "Agent-name collision scan: 0 collisions found across N global plugin agents."

4. The collision detection is a quality-of-life feature — it runs once per audit cycle as part of D4, not as a separate dimension.

**General notes for both subsections:**

- Per D-05: if plugin-dev is not installed, D4 falls back to existing manual cross-reference behavior. The installed_plugins.json parsing still works independently of plugin-dev (it's a local file), so the cross-reference check can still run even without plugin-dev. Only the collision detection needs plugin-dev installed (to find `agents/` directories in the plugin cache).
- The severity classification stays: BLOCK for broken refs, WARN for stale/missing refs, INFO for collision warnings.
- The section should document the approach in prose (as agent instructions), not embed full Python code — self-evolve is an agent, not a script. It should describe WHAT to check and HOW to check it, using Bash/Python as needed at runtime.
</action>
