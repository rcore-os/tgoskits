- Hook scripts use `PreToolUse` for tool-level gating and `SessionStart` for session-level gating
- SessionStart hooks in global settings.json use format: `{"hooks": {"SessionStart": [{"hooks": [{"type": "command", "command": "..."}]}]}}`
- Plugin-level hooks.json uses format: `{"hooks": [{"event": "SessionStart", "command": "python3 \"${CLAUDE_PLUGIN_ROOT}/scripts/validate-deps.py\""}]}`
- Hook scripts read CLI via `os.environ.get("CLAUDE_TOOL_INPUT", "")` with stdin fallback
- Error messages use "BLOCKED:" prefix followed by user action text on subsequent lines, output to stderr
- Exit 0 = pass, exit 1 = block

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| installed_plugins.json -> validate-deps.py | Untrusted JSON file read by hook script at session start |
| validate-deps.py -> stderr | Blocking messages printed to user's stderr stream |
| hooks.json -> validate-deps.py | Hook configuration controls which script runs at session start |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-FND-01 | Denial of Service | validate-deps.py JSON parsing | mitigate | Wrap `json.load()` in try/except; on JSONDecodeError print clear error and exit 1 (do not crash silently) |
| T-FND-02 | Elevation of Privilege | validate-deps.py file path | mitigate | Use `os.path.realpath()` when checking file existence to prevent symlink attacks |
| T-FND-03 | Spoofing | installed_plugins.json staleness | accept | Stale JSON masks plugin removal -- risk accepted as low (plugins rarely uninstalled mid-session); Layer 2 preamble provides second check at agent invocation |
| T-FND-04 | Denial of Service | validate-deps.py version parsing | mitigate | Defensive `parse_version()` wraps parsing in try/except, returns (0,) on any failure -- never crashes on malformed version strings |
</threat_model>
