<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent preamble -> Claude Code invocation | Preamble block is natural-language instructions, not code -- Claude interprets it |
| Frontmatter skills/tools -> validation list | Preamble lists items that must match Claude Code's actual context; mismatch means silent degradation |
| security-auditor removal -> agent body | Removing spawn references eliminates the only remaining code path that could silently fail |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-FND-05 | Spoofing | Preamble skills resolution | mitigate | Preamble instructs Claude to verify each `plugin:skill` resolves against installed plugins -- if skill not found, the agent must abort rather than silently skip |
| T-FND-06 | Denial of Service | security-auditor removal | accept | Removing spawn references means security audit capability is gone from scope. This is per D-09 (user decision). If security review is needed, it's a separate phase. |
| T-FND-07 | Tampering | Agent frontmatter YAML | mitigate | All frontmatter edits preserve valid YAML structure. Invalid YAML could cause Claude Code to silently ignore the agent. Verify with `grep` patterns that match YAML list element format. |
| T-FND-08 | Repudiation | Preamble abort consistency | accept | Preamble is natural-language instruction, not enforced code -- Claude MAY misinterpret. Accepting because: (a) Claude Code agents are naturally sequential processors, (b) Layer 1 hook already blocked missing plugins, (c) preamble is a defense-in-depth check. |
</threat_model>
